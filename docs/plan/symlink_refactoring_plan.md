# 심볼릭 링크(Symbolic Link) 연결 유지 검수 및 리팩토링 계획

**작성일**: 2026-04-29
**문서 상태**: 제안 및 실행 대기 (Plan)

---

## 1. 현황 및 문제점 분석

현재 워크스페이스 내에는 시스템의 SSoT(Single Source of Truth)인 `.ai/rules/rules.md`와 규칙 디렉토리를 각 AI 클라이언트에 주입하기 위해 **심볼릭 링크(Symbolic Link)** 가 다수 사용되고 있습니다.

**현재 식별된 심볼릭 링크 목록:**
- `CLAUDE.md` ➔ `.ai/rules/rules.md` (파일 링크)
- `.github/copilot-instructions.md` ➔ `.ai/rules/rules.md` (파일 링크)
- `.cursor/rules/` ➔ `.ai/rules/` (디렉토리 링크)
- `.windsurf/rules/` ➔ `.ai/rules/` (디렉토리 링크)

### 🚨 심볼릭 링크 유지에 대한 리스크 검수
심볼릭 링크는 관리가 편하다는 장점이 있으나, 현재 시스템 스케일에서는 다음과 같은 치명적인 결함을 유발할 수 있어 **유지하지 않는 것(제거)이 바람직**합니다.

1. **OS 권한 및 Git 형상 관리 충돌**: 
   - Windows 환경에서는 심볼릭 링크 생성 시 관리자 권한이 필요합니다. 
   - Mac/Linux와 Windows 사용자가 혼재된 환경에서 Git clone 시 심볼릭 링크가 단순 텍스트 파일(경로 문자열만 포함)로 변환되어버려 참조가 끊어지는 고질적인 문제가 있습니다.
2. **AI 에디터의 네이티브 인식 오류**:
   - Cursor나 Windsurf의 내부 인덱싱/RAG 엔진은 종종 심볼릭 링크로 연결된 디렉토리를 물리적 파일 시스템으로 인식하지 못해(Follow Symlink 옵션 누락 등), 핵심 룰이 무시되는 현상이 발생합니다.
3. **클라이언트 특화 설정의 불가능**:
   - 심볼릭 링크는 완전히 동일한 원본을 공유하므로, *'Cursor 전용 메타데이터'*, *'Copilot 전용 헤더'* 등을 각 환경에 맞게 커스텀하여 주입하는 것이 원천적으로 불가능합니다.
4. **기존 Sync 시스템과의 충돌**:
   - 현재 `mcp-cli`에는 `adapters/`와 `templates/`를 이용한 훌륭한 `install` 및 `sync` 로직이 이미 존재합니다. 굳이 OS 종속적인 심볼릭 링크에 의존할 필요가 없습니다.

## 2. 개선 계획 (Refactoring Plan)

심볼릭 링크를 전면 폐지하고, **물리적 파일 동기화(Physical File Synchronization)** 체계로 전환합니다.

### Step 1. 심볼릭 링크 제거
- 발견된 모든 심볼릭 링크(`.cursor/rules`, `.windsurf/rules`, `CLAUDE.md`, `.github/copilot-instructions.md`)를 삭제합니다.

### Step 2. MCP-CLI 기반 `sync` 파이프라인 고도화
- 각 환경의 어댑터(`claude_code.yaml`, `cursor.yaml` 등)와 베이스 템플릿을 기반으로, `.ai/rules/` 내의 SSoT 파일들을 **실제 파일로 복사하여 배포**하는 스크립트로 대체합니다.
- 동기화된 각 클라이언트의 룰 파일 최상단에는 다음 경고 문구를 주입합니다:
  > `[WARNING] 이 파일은 .ai/rules/ 에서 자동 생성된 동기화 파일입니다. 직접 수정하지 마세요.`

### Step 3. 전역 규칙(`.ai/rules/rules.md`) 갱신
- "설정 파일을 심볼릭 링크로 연결하라"는 기존 지침을 폐기합니다.
- 대신 "모든 룰 파일은 `.ai/rules`를 원본으로 하며, MCP 동기화 명령어(`mcp sync`)를 통해서만 각 클라이언트 영역에 물리적으로 덮어써야 한다"는 새로운 강제 규약을 명시합니다.

---

## 3. 기대 효과
- **크로스 플랫폼 호환성 확보**: 어느 OS에서 Git Clone을 받아도 설정이 깨지지 않습니다.
- **AI 인덱싱 안정성 100%**: 모든 클라이언트가 물리적인 마크다운 파일을 갖게 되어 파싱 누락이 사라집니다.
- **클라이언트별 맞춤 제어**: `.ai/rules`의 코어 룰을 유지하면서도 각 AI 환경의 특성에 맞게 템플릿을 주입할 수 있는 확장성을 확보합니다.
