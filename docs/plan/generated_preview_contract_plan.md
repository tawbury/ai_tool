# Generated Preview Contract Plan

## 개요

이 문서는 `.ai OS` Phase 8의 generated preview 계약을 정의한다. 현재 Phase 7 read-only sync runtime v0는 manifest, hash, marker, drift-stop, orphan-warning을 평가할 수 있지만 generated preview content를 만들지 않는다. 따라서 clean target과 future generated output을 비교해 `update` 후보를 완전하게 판단할 수 없다.

이번 계획은 preview contract만 정의한다. 런타임 코드, adapter generation, generated content 생성, sync apply, target mutation, manifest persistence는 구현하지 않는다.

## 목적

Generated preview의 목적은 다음과 같다.

- future generated output을 current target state와 비교한다.
- read-only dry-run에서 안전한 update candidate classification을 가능하게 한다.
- Phase 7 v0 hash policy와 provenance를 유지한다.
- future adapter generation, sync apply, rollback 설계 전에 output boundary를 명확히 한다.

Generated preview는 target에 쓰기 전에 만들어보는 "결과 후보"가 아니라, dry-run evaluator가 비교할 수 있는 read-only comparison input이다.

## 비목적

Generated preview는 다음이 아니다.

- adapter generation
- target mutation
- manifest persistence
- sync apply
- marker insertion
- marker repair
- marker deletion
- rollback execution
- worker execution
- workflow execution
- activation-driven sync selection

Preview가 존재해도 write authorization은 생기지 않는다.

## Preview Input Model

Preview provider가 미래에 입력으로 받을 수 있는 최소 모델은 다음과 같다.

```json
{
  "schema_version": "aios.generated_preview.input.v0",
  "entry_id": "entry_example",
  "manifest_entry": {
    "source_path": ".ai/rules/operations/sync.rules.md",
    "source_paths": null,
    "source_hash": "sha256:<source>",
    "target_path": "AGENTS.md",
    "sync_mode": "managed-block",
    "ownership": "mixed-boundary",
    "marker": {
      "marker_version": "0",
      "marker_style": "markdown-html-comment",
      "entry_id": "entry_example"
    }
  },
  "adapter": {
    "adapter_id": "codex-cli",
    "adapter_version": null
  },
  "context": {
    "activation_reference": null,
    "rule_context_reference": null
  },
  "hash_policy": {
    "version": "aios.hash_policy.v0",
    "algorithm": "sha256",
    "encoding": "observed-utf8-bytes",
    "line_endings": "preserve",
    "trailing_newline": "preserve",
    "managed_block_marker_lines": "exclude"
  }
}
```

필드 의미:

- `manifest_entry`: sync manifest entry의 정적 입력이다.
- `source_path`: 단일 source 파일이다.
- `source_paths`: future bundle source 입력 후보이며 v0에서는 선택 필드다.
- `source_hash`: manifest가 기록한 source hash다.
- `target_path`: preview 비교 대상 target이다.
- `sync_mode`: `whole-file`, `managed-block`, `mixed-boundary` 중 하나다.
- `marker`: managed block preview boundary를 설명한다.
- `adapter`: preview를 생성할 future adapter identity다. adapter 실행은 이 계획의 범위가 아니다.
- `context`: activation 또는 rule context 참조 후보이다. runtime selection은 아직 구현하지 않는다.
- `hash_policy`: generated bytes에도 적용될 hash policy다.

## Preview Output Model

Preview provider가 미래에 반환할 수 있는 최소 모델은 다음과 같다.

```json
{
  "schema_version": "aios.generated_preview.output.v0",
  "entry_id": "entry_example",
  "preview_available": true,
  "generated_content_kind": "managed-block",
  "generated_bytes_hash": "sha256:<generated-bytes>",
  "generated_target_hash": null,
  "generated_managed_block_hash": "sha256:<generated-managed-block>",
  "generated_metadata": {
    "adapter_id": "codex-cli",
    "adapter_version": null,
    "hash_policy": "aios.hash_policy.v0",
    "deterministic": true
  },
  "unavailable_reason": null,
  "provenance": {
    "source_paths": [".ai/rules/operations/sync.rules.md"],
    "source_hashes": {
      ".ai/rules/operations/sync.rules.md": "sha256:<source>"
    },
    "activation_reference": null,
    "rule_context_reference": null,
    "generated_by": "aios.generated_preview.v0"
  }
}
```

When preview is unavailable:

```json
{
  "schema_version": "aios.generated_preview.output.v0",
  "entry_id": "entry_example",
  "preview_available": false,
  "generated_content_kind": null,
  "generated_bytes_hash": null,
  "generated_target_hash": null,
  "generated_managed_block_hash": null,
  "generated_metadata": {},
  "unavailable_reason": "adapter-unavailable",
  "provenance": {
    "source_paths": [],
    "source_hashes": {},
    "generated_by": "aios.generated_preview.v0"
  }
}
```

필드 규칙:

- `preview_available`은 generated content가 비교 가능한지 나타낸다.
- `generated_content_kind`는 `whole-file`, `managed-block`, `mixed-boundary`, 또는 future explicit kind를 사용한다.
- `generated_bytes_hash`는 preview raw bytes에 대한 hash다.
- `generated_target_hash`는 whole-file target 비교에 사용하는 generated hash다.
- `generated_managed_block_hash`는 managed block inner content 비교에 사용하는 generated hash다.
- `generated_metadata`는 adapter identity, deterministic 여부, hash policy를 포함한다.
- `unavailable_reason`은 preview unavailable일 때만 채운다.
- `provenance`는 source paths, source hashes, activation/rule context reference, generated_by를 보존한다.

## Unavailable States

Preview unavailable reason은 최소 다음 값을 사용한다.

- `adapter-unavailable`: preview provider 또는 adapter identity가 준비되지 않음
- `source-missing`: source file 또는 source bundle이 존재하지 않음
- `unsupported-sync-mode`: 해당 sync mode에 preview contract가 없음
- `marker-invalid`: marker integrity가 valid가 아니어서 managed boundary를 신뢰할 수 없음
- `generation-disabled`: preview generation이 policy 또는 CLI option으로 비활성화됨
- `activation-unresolved`: preview에 필요한 activation/rule context reference가 해결되지 않음

Unavailable preview는 update candidate를 만들 수 없다.

## Dry-run Classification Extension

Future dry-run evaluator는 preview output이 있을 때 다음 순서를 따라야 한다.

1. Manifest schema validation을 먼저 수행한다.
2. Source existence와 source hash를 확인한다.
3. Target existence와 marker integrity를 확인한다.
4. Current target hash 또는 managed block hash를 manifest `target_hash`와 비교한다.
5. Target이 clean일 때만 generated preview hash와 비교한다.

Classification rules:

- clean target + generated hash differs -> read-only `update` candidate
- clean target + generated hash matches -> `skip`
- drifted target remains `drift-stop` even if generated hash exists
- invalid marker remains `conflict`
- missing source remains `conflict`
- preview unavailable must not create `update` candidate
- generated hash unavailable must keep existing Phase 7 v0 behavior

중요한 원칙:

- Preview는 drift-stop을 우회하지 않는다.
- Preview는 marker conflict를 복구하지 않는다.
- Preview는 ownership을 추론하지 않는다.
- Preview는 source of truth를 변경하지 않는다.

## Hash Policy

Generated preview는 Phase 7 v0 hash policy를 그대로 사용한다.

- observed UTF-8 bytes
- no CRLF/LF normalization
- preserve trailing newline and whitespace
- include BOM bytes if present
- use `sha256:<lowercase-hex>`
- exclude begin/end marker lines from managed block hash
- include managed block inner content exactly as generated

Generated preview는 future hash policy가 명시적으로 변경되기 전까지 line ending을 정규화하면 안 된다.

## Envelope v2 Mapping

Future sync dry-run envelope v2는 preview metadata를 다음 위치에 보존할 수 있다.

- `payload.results[].hashes.generated_target_hash`
- `payload.results[].hashes.generated_managed_block_hash`
- `payload.results[].details.preview`
- `payload.results[].details.preview_unavailable_reason`
- `messages[]` for preview unavailable warning/info or update candidate
- `meta.preview_policy`
- `meta.preview_provider`

Preview unavailable message policy:

- preview unavailable 자체는 command failure가 아니다.
- target이 clean이고 preview unavailable이면 기존 `skip` 또는 informational result를 유지한다.
- preview unavailable이 source missing, marker invalid 같은 기존 blocking condition에서 비롯된 경우 기존 blocking condition이 우선한다.

Update candidate message policy:

- `update` candidate는 warning이 아니라 informational planning result로 시작한다.
- mutation gate가 없으므로 `update` candidate는 write authorization이 아니다.
- message detail에는 generated hash, current target hash, manifest target hash, preview provenance가 포함되어야 한다.

## Adapter Boundary

Preview contract는 adapter identity를 참조할 수 있다.

Allowed:

- `adapter_id` 기록
- `adapter_version` 기록
- adapter determinism requirement 기록
- adapter output provenance 기록

Forbidden in this phase:

- adapter execution
- adapter generation
- adapter file write
- root adapter rewrite
- adapter-specific sync apply

Future adapter output은 deterministic하고 read-only여야 한다. 동일 input과 동일 adapter version은 동일 generated bytes hash를 만들어야 한다.

## Safety Rules

Generated preview safety rules:

- Preview must never authorize writes.
- Preview must not bypass drift-stop.
- Preview must not repair markers.
- Preview must not infer ownership.
- Preview must not create or update manifests.
- Preview must not insert anchors or markers.
- Preview must not execute workers or workflows.
- Preview must preserve source and generation provenance.
- Preview must use the same hash policy as dry-run comparison.
- Preview unavailable must fail closed into no-update-candidate.

## Future Implementation Slices

1. Contract-only now
   - 이 문서와 risk audit만 작성한다.
   - runtime code는 변경하지 않는다.

2. Optional preview fixtures next
   - generated preview input/output JSON fixtures를 추가한다.
   - 아직 provider는 구현하지 않는다.

3. Read-only preview provider later
   - deterministic fixture-backed provider 또는 explicit adapter stub을 구현한다.
   - target write는 계속 금지한다.

4. Dry-run update candidate extension
   - provider contract가 고정된 뒤 dry-run evaluator에 `update` candidate classification을 추가한다.
   - generated preview unavailable은 기존 Phase 7 behavior를 유지해야 한다.

## Phase 8 Entry Decision

Phase 8은 generated preview contract 설계로 시작한다.

이 단계는 adapter generation이나 sync apply를 앞당기지 않는다. 목적은 future generated output과 current target state를 안전하게 비교할 수 있는 read-only contract를 먼저 고정하는 것이다.
